## Phase 5: Admin API Endpoints (NEW)

**Goal**: JSON API so admin dashboard can use AJAX for dynamic data.

### New files:
- **`routes/admin_api.py`** — `admin_api_bp` blueprint at `/admin/api`, session-based auth returning JSON 401 on failure. ~20 endpoints:
  - **Users**: GET list (search/filter/paginate), GET detail, PUT toggle, PUT ban
  - **Sellers**: GET list, GET detail, PUT toggle, PUT approve, PUT reject
  - **Orders**: GET list (filter by status/seller/date, paginate), GET detail
  - **Analytics**: GET revenue (monthly), GET order-status breakdown, GET daily-orders, GET top-restaurants, GET overview
  - **Comments**: GET list (filter by restaurant/rating), DELETE
  - **Settings**: GET/PUT settings, CRUD categories, CRUD cities
